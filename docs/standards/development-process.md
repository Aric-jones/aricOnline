# 开发流程规范

## 标准流程

1. 先写或更新需求文档。
2. 再修改代码。
3. 再补变更日志。
4. 如模块职责变化，再更新模块说明。
5. 如产品范围变化，再更新项目总览。

## 必须先写文档的场景

- 新功能
- 修复明显行为问题
- 路由和菜单变化
- 权限和角色变化
- 接口变化
- 数据库变化
- 部署变化
- 配置变化

## 需求文档必须包含

- 背景
- 目标
- 非目标
- 影响范围
- 受影响文件
- 验收标准
- 验证计划
- 风险与回滚

## 变更日志必须包含

- 日期
- 标题
- 类型
- 需求文档链接
- 影响范围
- 验证方式
- 回滚说明
